# Render Setup For FactoryNerve Backend

This repo is now prepared for Render with:

- a backend Docker image that includes Tesseract OCR
- a `render.yaml` Blueprint
- a managed Postgres database definition
- startup migrations built into the container command

## Why Docker on Render

FactoryNerve OCR depends on OS-level Tesseract packages. Render recommends Docker when your app needs OS packages, and that applies here.

Official docs:

- https://render.com/docs/docker
- https://render.com/docs/blueprint-spec
- https://render.com/docs/deploys

## Files Added

- `render.yaml`
- `deploy/render/backend.Dockerfile`
- `.dockerignore`
- `scripts/generate_deploy_secrets.py`

## Recommended Architecture

- Frontend: Vercel on `https://factorynerve.online`
- Backend: Render web service
- Database: Render Postgres
- Browser API path: Vercel rewrites `/api/*` to the Render backend URL

You do not need to expose the backend on `api.factorynerve.online` on day one. The backend can live on its Render URL first, and Vercel can proxy to it.

## Render Deploy Steps

1. Open Render dashboard
2. Click `New +`
3. Choose `Blueprint`
4. Connect the GitHub repo:
   - `anonymous777999/factorynerve`
5. Render will detect `render.yaml`
6. Continue with the Blueprint deploy

## Required Secret Values

Before the backend can boot successfully, fill these secret env vars in Render:

- `DATA_ENCRYPTION_KEY`
- `SMTP_PASSWORD`
- `SMTP_FROM`
- one AI provider key:
  - `GROQ_API_KEY`
  - or `ANTHROPIC_API_KEY`
  - or `GEMINI_API_KEY`

The app validates these at startup, so leaving them blank will fail the deploy.

## Generate Safe Secrets

Run this locally:

```bash
python scripts/generate_deploy_secrets.py
```

It prints:

- `JWT_SECRET_KEY`
- `DATA_ENCRYPTION_KEY`

`JWT_SECRET_KEY` is already auto-generated by `render.yaml`, so the main one you need to paste manually is:

- `DATA_ENCRYPTION_KEY`

## Render Service Defaults In This Blueprint

- service name: `factorynerve-api`
- database name: `factorynerve-db`
- service plan: `free`
- database plan: `free`
- health check: `/observability/ready`

## Important Note About Free Plan

This config uses `free` as the safest cost-default.

For real client use, upgrade the backend service to at least `starter` so you avoid sleep/cold-start behavior.

## Backend URL After Deploy

Render will give you a URL like:

```text
https://factorynerve-api.onrender.com
```

Use that value in Vercel as:

```text
API_PROXY_ORIGIN=https://factorynerve-api.onrender.com
```

## Vercel Values To Set Next

In your Vercel project environment variables, set:

```text
API_PROXY_ORIGIN=https://factorynerve-api.onrender.com
NEXT_PUBLIC_API_PROXY_ORIGIN=https://factorynerve-api.onrender.com
```

Then redeploy Vercel.

## Domain Values For Backend

For the first working deployment, keep the Render backend on its default `.onrender.com` URL.

Later, if you want:

- `factorynerve.online` -> Vercel frontend
- `api.factorynerve.online` -> Render backend custom domain

you can add the Render custom domain after the service is live.

## Why Migrations Run In Container Startup

Render's pre-deploy command is not available on all plans. To keep the Blueprint simple and usable on the free plan, the Docker command runs:

```text
alembic upgrade head && uvicorn ...
```

That keeps first deploy simpler.
